## 즉시 조치 필요 (FAIL 2건)

1. **seed 미고정** (민정최종 + 한계점개선)
   - 두 노트북 모두 `np.random.seed(42)` 또는 `seed=42` 미설정
   - 영향: multi-start MLE 결과가 매번 다름 → 재현 불가
   - 조치: 노트북 셀 1(라이브러리)에 `np.random.seed(42)` 추가

## 검토 권유 (WARN 7건)

1. **ADF 사전검증** — `from statsmodels.tsa.stattools import adfuller` 추가 후 `BTC.diff().dropna()`에 대해 ADF 검정. p < 0.05면 정상.
2. **Ljung-Box 잔차 진단** — `from statsmodels.stats.diagnostic import acorr_ljungbox` 추가 후 표준화 잔차·잔차² 모두 점검.
3. **민정최종 Hessian 방법 명시** — 한계점개선처럼 `numdifftools.Hessian` 적용 또는 사용 안 함 사유 명시.
4. **한계점개선 분포 가정** — 정규분포 사용 중이면 BTC fat-tail에 부적합. Student-t로 통일 권장.
5. **alpha/omega bounds 노출** — 셀 상단에 모듈 변수로 추출 (`ALPHA_BOUNDS = (0.0, 0.5)` 등) 후 함수 호출.
6. **두 노트북 결과 비교 표** — 1차 수정 vs 한계점개선 결과 차이를 표로 정리해 보고서에 첨부.
7. **Aielli 2013 cDCC 적용 여부** — 두 노트북 모두 단변량 GARCH-X로 보임. 자산 간 동적 상관 분석은 별도 노트북에서 cDCC 적용 권장.

## 한계점 개선판의 학술적 정당성

`GARCH_분석_한계점개선.ipynb`의 세 개선사항은 모두 표준 문헌에서 권장:

| 개선 | 학술 근거 |
|---|---|
| ① alpha bounds 0.0 허용 | Hentschel (1995) "All in the family"에서 GARCH bounds 일반 논의. 경계 수렴 차단은 인위적 |
| ② omega 하한 강화 | Engle (1982) 원 ARCH 정의에서 omega > 0 필수. 1e-4는 수치 안정성 확보 |
| ③ numdifftools.Richardson | Lindfield & Penny (2012) "Numerical Methods" 표준 외삽법. 수치미분 SE 정확도 보장 |

→ 한계점개선판이 표준 문헌 권장값에 더 부합. **메인 분석 후보로 권장**.
