**문제**:
1. **B1**: `arch.arch_model` 은 **분산방정식 외생변수(GARCH-X)** 를 미지원이지만, 평균방정식 X 는 가능 (HARX 모델). 팀원은 이를 모르고 import 만 하고 커스텀 MLE 사용.
2. **B2**: alpha 가 하한 1e-6 에 경계 수렴. 변동성에 어제 충격이 영향 미친다는 GARCH 의 핵심 가정이 깨짐. t-stat=5000 같은 무의미 수치 발생.
3. **B3**: omega 가 0 으로 수렴. 분산 정의 위배 ($\sigma_t^2 \geq 0$).

### 2-2. 우리의 수정 — 두 갈래 접근

#### [방법 A] arch 라이브러리 표준 구현 (HARX-GARCH-StudentsT)

**모델 정의**:
$$r_t = \mu + \sum_{k=1}^{K} \gamma_k X_{k,t} + \epsilon_t$$
$$\sigma_t^2 = \omega + \alpha \epsilon_{t-1}^2 + \beta \sigma_{t-1}^2$$
$$\epsilon_t \mid \mathcal{F}_{t-1} \sim t_\nu$$

```python
from arch.univariate import HARX, GARCH, StudentsT

am = HARX(y, x=X, lags=0, constant=True)
am.volatility   = GARCH(p=1, q=1)
am.distribution = StudentsT()
res = am.fit(disp='off')
```

**장점**:
- 산업 표준 구현 → 표준오차·신뢰구간이 라이브러리 기본 (robust covariance) 으로 자동 계산
- 결과 신뢰도 확보

**단점**:
- 가설이 "변동성 → BTC 변동성" 에서 "**평균수익률**에 영향" 으로 변경됨

#### [방법 B] 커스텀 MLE FIXED (variance equation X — 팀원 가설 유지)

**모델 정의 (팀원 원본 유지)**:
$$\sigma_t^2 = \omega + \alpha \epsilon_{t-1}^2 + \beta \sigma_{t-1}^2 + \sum_{k=1}^{K} \gamma_k X_{k,t-1}$$

**수정**:
- **bound 강화**:
  - $\alpha \in [10^{-3},\ 0.5]$ (B2 해결)
  - $\beta \in [10^{-4},\ 0.999]$
  - $\omega \in [\hat{\sigma}^2_y \cdot 0.01,\ \hat{\sigma}^2_y \cdot 5]$ (B3 해결)
  - $\nu \in [2.5,\ 30]$
- **다중 초기값 grid search** (n_starts=8, β 시드 다양화)
- **표준오차**: Hessian 수치미분 + sandwich 근사 (단순 inverse-Hessian)

```python
def fit_garchx_grid(y, X_lag, K, n_starts=8):
    best_res = None
    best_nll = np.inf
    for x0 in starts:
        r = minimize(neg_log_lik_garchx, x0, args=(y, X_lag, K),
                     method='L-BFGS-B', bounds=bounds,
                     options={'maxiter': 500, 'ftol': 1e-8})
        if r.fun < best_nll and np.isfinite(r.fun):
            best_nll = r.fun
            best_res = r
    return best_res
```

### 3-2. 우리의 수정 — 윈도우 sensitivity

**4 윈도우 × 6 이벤트 × 5 자산 = 120 검정** 동시 실행.

```python
WINDOWS = [3, 5, 10, 17]
for window in WINDOWS:
    for ev in EVENT_DATES:
        for asset in ASSETS:
            # AR 산출 + BMP + Block Bootstrap
            ...
```

**모델 사양 (팀원 v4 유지)**:
- BTC: $E[R_t] = \alpha_{BTC} + \beta_{BTC} R_{NASDAQ,t}$
- Gold, DXY: $E[R_t] = \bar{R}$ (CMRM)
- TLT, NASDAQ: $E[R_t] = \alpha + \beta R_{SP500,t}$

**검정**:
- **BMP** (Boehmer-Musumeci-Poulsen): 추정기간 σ로 표준화한 SAR
  $$SAR_t = \frac{AR_t}{\hat{\sigma}_{est}}, \quad t = \frac{\overline{SAR}}{\sqrt{Var(SAR)/T}}$$
- **Circular Block Bootstrap**: 추정기간 AR 에서 block=5 길이로 5,000 번 샘플링하여 CAR 분포 생성. 자기상관 보존.

## 6. 최종 결론 (수정 후)

### 가설 검증
**BTC 는 일관된 안전자산이 아니다.** Baur & Lucey (2010) 3 조건 어느 것도 6 이벤트에서 일관되게 충족하지 않음.

### 새롭게 밝혀진 사실
1. **시장공포(VIX)가 BTC 변동성을 가장 강하게 설명** — 팀원 결과(비유의)와 정반대
2. **공식 GPR 은 변동성을 낮추는 방향** (시장공포 통제 시) — 직관 반대
3. 팀원의 "이스라엘-하마스 Safe Haven" 결론은 **윈도우 의존** — robust 하지 않음
4. 팀원 BTC 로직은 변동성을 15.5% 축소 → 모든 추정량 편향 가능

### 발표용 한 줄
> "팀원 원본의 4가지 critical 결함을 수정한 결과, BTC 는 안전자산이 아니라는 결론은 강화되었으나, 그 메커니즘은 **지정학 리스크가 아니라 시장 공포 채널**임이 새롭게 드러났다."
