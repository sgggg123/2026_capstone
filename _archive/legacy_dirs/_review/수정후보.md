### 4.5 Edit_mj/GARCH_분석.ipynb (33셀, 902 줄) — 가장 주의 깊게 볼 노트북

| 항목 | 내용 |
|---|---|
| 목적 | GARCH(1,1)-X 5모델 비교, GPR γ 계수 유의성 |
| 입력 | `master_data.csv` (216행 → 결측 제거 210행) |
| 출력 | `garch_model_comparison.csv`, `garch_gamma_results.csv`, `garch_conditional_volatility.csv` + 3 PNG |
| 모델 사양 | `h_t = ω + α·ε²_{t-1} + β·h_{t-1} + Σ γ_k·X_{k,t-1}` (X 는 t-1 lag, t-분포 잔차 nu) |
| 5 모델 외생변수 | M1: GPR_zscore / M2: GPR_custom / M3: VIX+F&G / M4: 공식+심리 / M5: 커스텀+심리 |
| **구현 방법** | `arch.arch_model` import 만 하고 사용 안 함. **scipy.optimize.minimize 로 커스텀 MLE 직접 작성**. SE 는 Hessian 수치미분(eps=1e-5) |
| 결과 | M2 가 AIC 최저(1087.05), γ_GPR_custom=+4.001 (p=0.192 비유의) |
| 본인 자인한 한계 | ① alpha=0 경계 수렴 (M1,3,4,5) ② omega=0 수렴 (M3,4) ③ Hessian 수치미분 오차 ④ 표본 210<권장 300 ⑤ 비대칭성 미반영 ⑥ 선형성 가정 ⑦ lag 1 외 미검정 |

## 7. 다음 단계 (Phase 2 진입 시 우선순위 제안)

1. **B1 (GARCH 라이브러리 재검증)** — 가장 critical. arch_model 의 외생변수 옵션이 진짜 안 되는지 확인 후, 안 되면 rpy2-rugarch 또는 sandwich SE 로 우회. 결과 안정성 향상 폭 가장 큼.
2. **B2/B3 (alpha=0, omega=0 경계)** — B1 해결되면 자동 완화 가능성 높음. 안 되면 다중 초기값 + bound 조정.
3. **C1 (이벤트 윈도우 sensitivity)** — ±3, ±5, ±10, ±17 표 만들어서 결과 안정성 확인. 가장 빠른 작업.

5. **D1 (Quantile β_τ vs β_0.50 Wald test)** — 한 줄 추가로 끝남.
