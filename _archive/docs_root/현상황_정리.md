## 😵 헷갈렸던 5가지 포인트 — 정확히 정리

### 1. "이게 뭐 하는 거야?"

**검증 시스템 = 분석 코드가 학술 논문 표준대로 됐는지 자동으로 검사하는 도구.**

비유하면 "맞춤법 검사기"인데, 대상이 통계 분석 코드.

### 2. "새 데이터로 왜 안 돌렸어?"

**돌릴 필요 없음. 우리가 한 건 분석이 아니라 분석 절차 검증.**

- 데이터를 새로 돌려도 → 수치만 바뀜
- 절차 위반(추정창 95일, Placebo 누락 등)은 → 그대로 남음
- 그래서 **절차 먼저 고치고, 데이터 다음**

### 3. "검증하려면 뭐가 있어야 해?"

**두 가지뿐.**

| 필요한 것 | 상태 |
|---|---|
| 분석 코드 (노트북·스크립트) | ✅ 이미 있음 (MJ가 만든 노트북 + 본인이 만든 스크립트) |
| 검증 도구 (`.claude/` 폴더) | ✅ 방금 만듦 |

### 4. "팀원 코드는 어떻게 검증해?"

**이미 등록 완료. 명령만 누르면 됨.**

| 담당자 | 명령어 |
|---|---|
| 본인 (이벤트 스터디) | `/verify-event-study` |
| GARCH 담당 | `/verify-dcc-garch` |
| 분위회귀 담당 | `/verify-quantile-reg` |
| GPR 담당 | `/verify-gpr-hybrid` |
| 전체 한 번에 | `/verify-all` |

catalog.json에 7개 방법론 다 등록돼있어서 추가 작업 필요 없어.

### 5. "노트북, 터미널, 대시보드 뭘 보여줘?"

**노트북 하나만.**

- 노트북에 catalog 로드, 검증 결과, 인용까지 다 들어있음
- 터미널 슬래시 명령은 시연 안 함 (노트북이 더 깔끔)
- 대시보드는 별개 (이번 작업 범위 아님)
