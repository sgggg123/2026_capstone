## 1. 구축 배경 (Why)

본 프로젝트는 Event Study (MacKinlay 1997), DCC-GARCH (Engle 2002), Quantile Regression (Koenker & Bassett 1978), Hybrid GPR (Caldara & Iacoviello 2022 + FinBERT + OECD PCA), Stationary Bootstrap (Politis & Romano 1994), Baur & Lucey (2010) safe-haven 프레임워크 등 **7개 이상의 학술 표준 방법론**을 결합한다.

문제는 코드/결과/해석이 표준 절차와 일치하는지 매번 수동으로 확인해야 했다는 점이다. 결론을 진술할 때마다 출처 인용을 의식적으로 첨부해야 했고, 누군가 파라미터를 바꾸면 학술 표준 위반 가능성을 즉시 감지할 장치가 없었다.

**해결**: Claude Code의 skills + references 기능을 활용해 (1) 학술 표준 카탈로그, (2) 자동 검증 명령, (3) 인용 자동 첨부 시스템을 한 묶음으로 구축했다.

## 5. 핵심 자산: `catalog.json`

7개 방법론 각각에 대해 다음 구조로 통합 관리:

```json
{
  "event_study": {
    "paper": "MacKinlay (1997) JEL 35(1):13-39",
    "key_assumptions": ["추정창 ≥120거래일", "이벤트 클러스터링 없음", ...],
    "recommended_params": {"est_window_days": [120, 250], "event_window_days": [-5, 5], ...},
    "red_flags": ["est_window < 100", "Bootstrap 미사용", "Placebo 누락", ...],
    "source_files": ["Edit_mj/이벤트_스터디_v2.ipynb", ...],
    "result_files": ["_review/results/event_study_sensitivity.csv", ...],
    "param_regex": {...},
    "citation_kr": "MacKinlay (1997, JEL 35:13-39)에 따른 표준 이벤트 스터디 절차",
    "citation_bibtex": "@article{mackinlay1997event,...}"
  }
  // dcc_garch, quantile_reg, gpr_hybrid, bootstrap, multiple_testing, safe_haven 동일 구조
}
```

이 한 파일이 **모든 검증 스킬의 기준점**이다. 논문이 바뀌거나 권장값이 갱신되면 여기만 고치면 됨.

## 7. CLAUDE.md 핵심 지침

1. 통계 결론 진술 시 `.claude/references/catalog.json`의 `citation_kr` **자동 첨부**
2. τ, 윈도우, 블록크기, α* 등 핵심 파라미터 변경 시 변경 전후를 커밋·PR에 명시
3. 새 방법론 도입 시 `references/<method>.md` 코드보다 **먼저** 작성
4. "BTC는 (강한/약한) safe haven이다/아니다" 결론 작성 전 `/verify-safe-haven` 실행
5. 노트북 셀 첫 줄에 `# METHOD: <name>` 주석 (검증 스킬의 셀 단위 매핑용)
6. 응답 언어: 한국어

## 12. 결론

본 구현물은 **단일 truth source(`catalog.json`) + 분산 실행자(9개 슬래시 스킬) + 자동 인용 지침(CLAUDE.md)** 의 3축으로 구성된다. 새 방법론 추가, 표준 논문 갱신, 검증 항목 확장이 모두 카탈로그 1개 파일 수정으로 전파되며, 코드 변경 시점마다 `/verify-all` 한 줄로 학술 표준 위반을 자동 감지한다.

본 시스템 자체가 첫 실행에서 `/verify-event-study`로 **Placebo 검증 누락 1건 + 검토 권유 4건**을 자동 검출함으로써 작동의 실효성을 입증했다.
