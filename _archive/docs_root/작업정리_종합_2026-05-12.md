## 🔧 어떻게 작동하는지 (요약)

### 검증 흐름
```
[1] catalog.json 로드
       ↓
[2] source_files 정규식 자동 추출
    (변수, 함수 인자, 키워드)
       ↓
[3] catalog의 red_flags · recommended_params와 대조
       ↓
[4] PASS / WARN / FAIL 객관 판정
       ↓
[5] verification_reports/ 에 결과 자동 저장
       ↓
[6] 결론에 학술 인용 자동 첨부
```

### 슬래시 명령 (팀원 사용법)
```
/verify-event-study     ← 이벤트 스터디 (본인)
/verify-dcc-garch       ← GARCH 담당
/verify-quantile-reg    ← 분위회귀 담당
/verify-gpr-hybrid      ← GPR 담당
/verify-safe-haven      ← 최종 판정
/verify-all             ← 7개 한 번에
/cite-method <name>     ← 인용 즉시 출력
```
